  Each curandState_t is one independent RNG. curand_init(seed, sequence, offset, &state) sets up a unique RNG sequence for    
  that state — the sequence number (which you set to the array index idx) is what makes each state independent from the       
  others. Then curand_uniform advances that state and returns one float.                                                      
                                                                                                                              
  For your two arrays: X needs I*J*K random numbers, U needs J*I. The simplest approach is to allocate I*J*K states (enough   
  for X), then reuse a subset of them for U.
                                                                                                                              
  The generateRandomNumbers call for U already does this — it passes (J, I, 1) as dimensions, so the kernel computes idx = j +
   i*J, which only touches states at indices [0, I*J-1]. Those are all within the I*J*K allocation.
                                                                                                                              
  The one subtlety: those states were already advanced once (to fill X's first slice), so U gets the second number from each  
  of those sequences rather than the first. For a benchmark this is completely fine — the values are still pseudo-random.
                                                                                                                              
  So the fix for line 113 is just:                                                                                            
  CHECK_CUDA(cudaMalloc(&dev_states, (long)I*J*K*sizeof(curandState_t)));
